ЧАСТЬ А
1. Создайте публичный репозиторий на GitHub или Bitbucket и добавьте файл
.gitignore. В этом репозитории будут храниться все лабораторные работы,
которые будут разрабатываться в дальнейшем.
2. Авторизуйте открытый ключ на GitHub или Bitbucket.
3. Клонируйте созданный репозиторий на свой компьютер.
4. Создайте новую ветку с именем "labs_git" и перейдите на нее. Эта ветка
будет основной для сдачи задания.
5. Создайте новую ветку с именем "task1" и перейдите на нее. Эта ветка будет
основной для выполнения задания.
6. Находясь на ветке "task1", создайте папку labs.
7. В папке labs создайте файл с именем lab1.txt и поместите в него половину
текста задания лабораторной работы.
8. Сделайте коммит с именем "add first half of labs" и отправьте его на сервер.
9. Убедитесь, что в репозитории на GitHub (или Bitbucket) сейчас есть файл
lab1.txt и он содержит половину текста задания.
10. Используя веб-интерфейс GitHub (или Bitbucket), добавьте вторую
половину текста в файл lab1.txt и сделайте коммит.
11. В вашем локальном репозитории извлеките изменения и убедитесь, что
изменения, сделанные во втором коммите, доступны, и вы сможете увидеть
полный текст задания.
ЧАСТЬ Б
1. Настройте файл .gitignore так, чтобы скрыть файлы с расширениями .db,
.log, .iml и папки с именами target, bin, .idea.
2. Создайте новую ветку с именем "task2", ответвившись от "task1", и
перейдите на нее.
3. На ветке "task2" создайте файл testGit.txt в папке labs.
4. В файл testGit.txt добавьте строку "first commit". Сохраните и закоммитьте
изменения.
5. В файл testGit.txt добавьте строку "second commit". Сохраните и
закоммитьте изменения.
6. Вмержите ветку "task2" в "task1".
7. Вернитесь на ветку "task2" и добавьте строку "third commit". Сохраните и
закоммитьте изменения.
8. Перейдите на ветку "task1" и добавьте строку "fourth commit". Сохраните и
закоммитьте изменения.
9. Вмержите ветку "task2" в "task1", разрешив конфликт таким образом, чтобы
все строки остались в правильном порядке.
10. Запушить все ветки на https://github.com (https://bitbucket.org).
11. Предоставьте уровень доступа "READ" к удаленному репозиторию
пользователю vgkozlov.
12. Открыть пулл реквест на ветку labs_git.
